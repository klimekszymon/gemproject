# AI Agent Skill Development: The Official Learning Path

Welcome to the Team Platform Skill Registry. This repository is designed to take you from a beginner to a **Platform Architect**.

---

## 🛤️ The Curriculum (Status: ✅ Completed)

### 01: Foundations (The "How") - ✅
*   **[01-intro-to-skills.md](./01-foundations/01-intro-to-skills.md)**: What are skills, and why do we use them?
*   **[02-initializing-skills.md](./01-foundations/02-initializing-skills.md)**: Using standard tools to scaffold your first skill.
*   **[03-dynamic-scripts.md](./01-foundations/03-dynamic-scripts.md)**: Making your skills deterministic with Python, Node, or Bash.
*   **[04-progressive-disclosure.md](./01-foundations/04-progressive-disclosure.md)**: Managing the context window with `references/`.
*   **[05-packaging-and-distribution.md](./01-foundations/05-packaging-and-distribution.md)**: Creating `.skill` files for sharing.

### 02: Advanced Automation (The "Scale") - ✅
*   **[06-session-closers.md](./02-advanced-automation/06-session-closers.md)**: Building skills that summarize and "handoff" your work.
*   **[07-skill-creator.md](./02-advanced-automation/07-skill-creator.md)**: Building meta-skills (skills that build skills).
*   **[08-registry-automation.md](./02-advanced-automation/08-registry-automation.md)**: Automating the central registry index.
*   **[09-quality-gates.md](./02-advanced-automation/09-quality-gates.md)**: Validating skill integrity in CI/CD.
*   **[10-asset-management.md](./02-advanced-automation/10-asset-management.md)**: Managing templates, boilerplate, and binary assets.

### 03: Mastery Ops (The "Architecture") - ✅
*   **[11-orchestration-and-chaining.md](./03-mastery-ops/11-orchestration-and-chaining.md)**: Designing multi-skill handoffs and workflows.
*   **[12-secrets-and-environments.md](./03-mastery-ops/12-secrets-and-environments.md)**: Securely managing API tokens and environment-specific logic.
*   **[13-prompt-engineering.md](./03-mastery-ops/13-prompt-engineering.md)**: Mastering `SKILL.md` instructions and few-shot grounding.

### 04: Enterprise Orchestration (The "Ecosystem") - ✅
*   **[14-skill-registry-governance.md](./04-enterprise-orchestration/14-skill-registry-governance.md)**: Versioning, deprecation, and registry synchronization.
*   **[15-telemetry-and-performance.md](./04-enterprise-orchestration/15-telemetry-and-performance.md)**: Tracking skill usage, success rates, and token efficiency.
*   **[16-inter-skill-communication.md](./04-enterprise-orchestration/16-inter-skill-communication.md)**: Advanced chaining via shared state and JSON contexts.

### 05: Domain-Specific Architectures (The "Experts") - ✅
*   **[17-devops-mastery-with-skills.md](./05-domain-specific-architectures/17-devops-mastery-with-skills.md)**: Skills as CI/CD triggers, monitoring agents, and IaC auditors.
*   **[18-frontend-ui-generation.md](./05-domain-specific-architectures/18-frontend-ui-generation.md)**: Scaffolding components from design tokens and CSS auditing.

### 06: Autonomous Evolution (The "Future") - ✅
*   **[19-self-healing-skills.md](./06-autonomous-evolution/19-self-healing-skills.md)**: Detecting errors and proposing code fixes to skills.
*   **[20-the-meta-registry-architect.md](./06-autonomous-evolution/20-the-meta-registry-architect.md)**: Building skills that architect and scaffold entire new learning tracks.
