# Instructions
1. **Sanity Check**: Before closing, run any relevant project checks (e.g., `npm run lint` or `npm test`) if applicable to ensure the session ends on a "passing" state.
2. **Execute Automation**: Run the `scripts/end-session.sh` script to update the persistent context and commit today's work.
3. **Verify Persistence**: Read the last 10 lines of `docs/ai_context.md` (or the file specified in the script) to ensure the summary and next steps were correctly written.
4. **Final Handoff**: Provide the user with a 1-sentence "cliffhanger" summary that reminds them exactly where to pick up in the next session, based on the "Next Steps" generated by the script.

## Why use this skill?
Unlike a basic handoff, this skill ensures that your project state is **hardened to disk** and versioned in Git, making it accessible to any AI agent in the future.
